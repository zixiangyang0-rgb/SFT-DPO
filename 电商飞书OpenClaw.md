# 项目经历 · AI 卖家助手（电商飞书 OpenClaw）

**公司背景**：工程机械配件外贸出口
**岗位**：AI 应用开发

## 一、做了什么

我参与建设了一套面向工程机械配件外贸卖家的 AI 助手平台，基于 OpenClaw 框架，将 AI 能力接入飞书工作群，让运营人员在飞书里直接用自然语言查询销售、库存、客户、订单等业务数据，无需切换后台系统。

**2. Skill 封装与业务对接**

传统模式下，每增加一个数据查询需求都需要工程师写代码、发版本，周期以周计。我采用 Markdown Skill 机制替代这套流程——运营人员或业务人员只需编写一份 SKILL.md 文件，放入 skills 目录即可生效，响应周期压缩到分钟级。

具体封装了四个平台级 Skill：

- **销售查询（sales-query）**：对接订单 API，卖家在飞书问"今天卖了多少"，AI 自动调接口、解析数据、返回结构化答复，并支持退货分析和趋势对比
- **商品排名（product-ranking）**：对接商品 API，按销量、库存周转等维度返回热销榜、滞销品预警，帮助运营快速识别哪些配件卖得好、哪些积压
- **库存预警（inventory-alert）**：对接库存 API，设定低水位阈值后，AI 自动监控并在飞书推送补货建议，避免断货影响外贸交期
- **客户洞察（customer-insight）**：对接客户 API，统计新客/回头客比例、地域分布，辅助销售策略调整

每个 Skill 本质上是把 API 接口用自然语言包装了一遍——业务接口没变，但使用门槛从"会操作后台"变成了"会说话"。

**3. 主动通知与多 Agent 分工**

传统的 AI 助手是"你问我答"，但外贸运营场景中，运营人员不会主动想起去查库存、看异常。我利用 OpenClaw 内置的 Cron 系统，将被动响应升级为主动推送：配置每 30 分钟自动检查库存状况，发现低库存或断货风险立即推送飞书通知；每日早报自动汇总昨日销售概况、退货率和补货提醒。运营人员无需任何操作，数据主动出现在群里。

此外，针对售后客服和销售运营两类角色，配置了不同的 Agent：售后 Agent 加载退款处理和工单查询 Skill，人格定义为"耐心专业、优先安抚情绪"；销售 Agent 加载销售分析和商品排名 Skill，人格定义为"数据优先、结论先行"。两类角色在同一个飞书体系下各司其职，互不干扰。

**4. 安全体系设计**

AI 助手通过 bash 执行 curl 命令调用业务 API，这给了 AI 灵活性，但也带来风险。我在部署层面构建了多层防护：命令白名单限制 AI 只能使用 jq、grep、curl 等指定程序；环境变量黑名单拦截 LD_PRELOAD、NODE_OPTIONS 等注入攻击；所有商家上传的 Skill 文件上线前经过安全扫描器检测，识别挖矿代码、恶意混淆等模式；每个商家的 API 凭证通过 AWS Secrets Manager 管理，配合 IAM Policy 实现基础设施级别的访问隔离，任何商家只能访问自己的数据。这套机制确保了 AI 执行过程的可知可控。

## 三、电商平台 API 对接规划

项目 PoC 阶段使用 Mock API 验证了框架能力，下一阶段计划对接真实电商平台 API。以下是针对工程机械配件外贸出口场景评估的主流平台：

**Tier 1 · 最优先对接（工程机械 B2B 外贸直接相关）**

- **阿里巴巴国际站（Alibaba.com）**：B2B 外贸出口首选平台，API 通过淘宝开放平台（TOP）接入，覆盖商品发布（`alibaba.icbu.product.add`）、订单管理、物流轨迹等核心接口，认证方式为 AppKey + HMAC 签名 + OAuth 2.0 授权。工程机械配件在该平台询盘增长 45%、订单增长 127%，配件出口占行业 40%，与项目场景高度匹配
- **亚马逊 Seller Central（SP-API）**：面向北美/欧洲终端买家的 B2C 配件零售，2026 年新版 Orders API v2026-01-01 支持单次请求拉取全量订单信息，减少调用次数，覆盖订单/库存/销售报告/商品评价等接口

**Tier 2 · 场景补充**

- **敦煌网 DHgate**：小额 B2B 跨境出口，北美买家为主，提供 open.dhgate.com 开放平台，OAuth 2.0 认证
- **Shopify**：有独立站的配件卖家，GraphQL Admin API，2026 年全面 GraphQL-First 架构
- **Shopee / 速卖通**：REST API，东南亚/俄语市场补充

**降本路径**

若需快速对接多平台，建议通过 API2Cart（统一 API，一次接入 60+ 平台）或 Apideck 做中间层标准化，OpenClaw 只需维护一个适配器即可支持亚马逊、Shopify、敦煌、eBay 等多个平台，大幅降低对接维护成本。

## 四、功能与业务如何对接

外贸出口场景的核心痛点是：业务数据分散在多个平台（电商后台、ERP、物流系统），运营人员每天要在多个系统之间来回切换，既费时又容易出错。

AI 助手做的事情，就是把这些系统"藏起来"，只留一个对话入口。卖家在飞书里说一句话，AI 背后做的事情是：识别意图 → 调用对应 Skill → 执行 curl 请求业务 API → 解析返回数据 → 用自然语言回答。整个过程对运营人员透明，他不需要知道 API 是什么、参数怎么填，只需要说"这周哪个配件卖得最差"。

同时，工程机械配件外贸有一个显著特点——SKU 数量多、型号规格复杂、库存周转直接影响交期。库存预警 Skill 直接对应该需求：当库存低于安全水位，自动推送补货建议，帮助运营提前行动，避免因配件断货导致海外客户订单延误。

定时主动推送则解决了"想起来才查"的问题。对于每天盯几十个数据指标的运营人员，AI 主动把异常推过来，比被动查询效率提升一个量级。
