# AI 应用开发工程师 · 项目经历

> 适用岗位：AI 应用开发
> 公司背景：工程机械配件外贸出口（主营挖掘机零部件、液压密封件、紧固件，出口东南亚、中东、非洲等海外市场）
> 项目名：数字永生蒸馏框架（immortal-skill）
> 字数目标：约 1000 字

---

## 一、业务背景与真实痛点

公司做工程机械配件外贸出口，业务场景比较特殊：

- **配件种类杂**：同样一个"垫片"或"密封圈"，不同客户询的型号、材质、认证标准都不一样，销售要记住几百个 SKU 的差异化参数；
- **经验全在老业务脑子里**：客户谈判怎么压价、哪个区域用什么策略、哪些认证能豁免、丢单的原因是什么——这些全靠老业务口口相传或者埋在微信聊天记录里；
- **人员一走就断档**：老业务离职，带走客户也带走经验；新业务上岗，光熟悉产品参数就要三个月，谈判更是无从下手；
- **跨部门协同散**：采购、关务、仓库、海外代理之间大量信息通过微信和邮件传递，没有结构化沉淀，查一个历史订单要翻半天记录。

我作为 AI 应用开发工程师，核心任务就是把**散落在微信、邮件、飞书里的隐性经验，变成 AI 可以调用的知识资产**。

## 二、我具体做了什么

### 1. 多平台数据采集——解决"数据从哪来"

外贸业务的数据极度分散：
- 海外客户的询盘、议价、成交记录在邮件里；
- 老业务的谈判策略和判断在微信私聊里；
- 售后故障处理在飞书工单里；
- 采购清单和供应商沟通在企业微信里。

我设计了一套**插件化的采集框架（BaseCollector 抽象模式）**，为每个平台写独立的采集插件，统一输出为标准 JSON 语料包。主要攻克了三个技术难点：

- **微信本地 SQLite 数据库解析**：业务人员导出的微信聊天记录是.db 文件，我解析其表结构，提取消息正文、发送者、时间戳，转换为结构化数据；
- **飞书 Open API 接入**：获取飞书应用授权后，通过分页接口拉取工单记录，处理中文编码和长文本截断；
- **邮件归档解析**：处理 .eml 和 .mbox 格式，提取主题、正文、附件名，过滤掉自动推送类邮件。

所有数据走本地存储，不上云，符合外贸业务对客户信息的合规要求。同时做了自动化脱敏，客户姓名、合同金额等敏感字段替换为代号。

### 2. 四维人格蒸馏——解决"经验怎么提炼出来"

光采集数据没用，关键是怎么把老业务脑子里的"隐性经验"提炼成 AI 能用的结构化知识。我设计了一套**四维蒸馏引擎**，把一个人的经验拆成四个维度分别提取：

| 维度 | 翻译成外贸业务含义 | 典型产出 |
|------|------------------|---------|
| 程序性知识 | 怎么做这件事 | 报价 SOP、售后故障排查流程、认证材料准备清单 |
| 互动风格 | 怎么跟客户说话 | 议价话术、催款语气、投诉安抚方式 |
| 记忆经历 | 经历过什么关键事件 | 大客户攻关复盘、丢单原因、突发事故处理 |
| 性格价值观 | 这个人做决策的特点 | 保守还是激进、风险偏好、价格底线 |

每个维度的提取结果都标注了**证据等级**：
- `verbatim`（原话引用）：客户原话或业务本人直接说的话；
- `artifact`（文档推断）：从邮件、合同、工单里推导出来的信息；
- `impression`（主观判断）：从行为模式推断，但未经当事人确认。

对于同一件事不同来源说法矛盾的地方，我没有强行消解，而是记录在 `conflicts.md` 供业务人员裁定——因为真实业务本来就是矛盾的，强行统一反而会丢失信息。

### 3. 推理链路与业务对接——解决"AI 输出能不能用"

蒸馏出来的知识要真正服务业务，还需要一层加工：

- **用户纠正处理器**：业务人员发现某条提取结果不对，可以修正，系统只更新相关条目而非全量重跑，生成变更报告后自动更新版本；
- **版本快照系统**：每次蒸馏完成后生成快照，支持回滚到任意历史版本，防止误操作导致数据丢失；
- **证据溯源**：每条 AI 输出都可以回溯到原始语料，业务人员在引用 AI 建议时能说清楚依据是什么。

### 4. CLI 工具链——让非技术人员也能用

最终封装成一套命令行工具，支持：
- `platforms` 查看支持的平台列表；
- `collect` 按平台拉取数据；
- `snapshot` 创建版本快照；
- `rollback` 安全回滚；
- `stamp` 生成文件指纹。

业务人员不需要写代码，在终端执行几个命令就能完成数据采集和 Persona 更新，降低了 AI 工具的使用门槛。

## 三、功能实现的业务用途

这套框架在三个场景里直接产生价值：

**场景一：新人快速上手**
老业务离职后，新人入职只需要在 AI 助手对话框里 @ 对应角色，就能问到：
- "阿尔及利亚客户询这个型号的轴承，交期和认证怎么回？"
- "上次跟这个客户谈判卡在哪里了？"

原来三个月才能上手，现在四周能独立跟客户沟通。

**场景二：跨时区客户响应**
海外客户发来询盘，AI 可以先基于历史对话给出参考报价和推荐话术，业务人员确认后再回复，不用等老业务在线。

**场景三：经验资产留痕**
老业务在职时就可以把他的经验蒸馏成数字分身，离职后经验不跟着人走，新人随时可以调用，降低了人员流动对业务的冲击。

## 四、技术能力沉淀

通过这个项目，我完整跑通了 AI 应用从数据采集、语义提炼到业务落地的全链路：

- 多平台 API / 本地数据库混采，本地优先存储与脱敏；
- Prompt 蒸馏工程：四维拆解、证据分级、矛盾保留；
- Agent Skill 标准封装：打包、加载、版本管理；
- 把 AI 能力翻译成业务团队用得上的工具，不是概念，是每天真有人在用的东西。
