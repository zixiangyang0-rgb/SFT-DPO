# 主流模型微调方案面试话术（1000字速记版）

> 适用：NLP/大模型算法岗、AI 应用岗；核心套路：**讲人话 + 讲对比 + 讲选择理由**。

---

## 一、30 秒开场（先说全貌）

> "我对主流微调方案的理解可以浓缩成一句话：**Pre-train 决定智商，SFT 决定'懂规矩'，DPO/RLHF 决定'做好人'**。SFT 教的是'说什么'，DPO 教的是'哪种说法更好'。我平时做项目主用 SFT + DPO 两阶段，简单任务直接 SFT 一次，复杂任务上 ORPO（把 SFT 和 DPO 合并成一步，省显存还不用 Reference Model）。**工具链上，原理研究用 Hugging Face TRL，个人单卡用 Unsloth（快 2-5 倍），业务方零代码用 LLaMA-Factory，多卡分布式才上 DeepSpeed。**"

---

## 二、SFT / DPO / PPO / ORPO 怎么选？（方案选型题）

面试官最常问的对比题，**直接套表回答**：

| 方案 | 本质 | 显存 | 难度 | 适用 |
|---|---|---|---|---|
| **SFT** | 喂 (Instruction, Output) 标准答案 | 低 | 最低 | 教模型"按规矩说话" |
| **DPO** | 喂 (Chosen, Rejected) 让模型挑好的 | 中（要 Reference Model） | 中 | 教模型"什么是更好的回答" |
| **PPO** | 强化学习，要训 Reward Model + Critic | 极高 | 极难 | 大厂旗舰产品，**个人开发者基本不用** |
| **ORPO** | 把 SFT 和 DPO 写进同一个 Loss | **低（不要 Reference Model）** | 中 | 单卡显存吃紧 + 想少训一轮 |

**金句**："**SFT 是铺地基，DPO 是装修，PPO 是把房子推倒重盖**。地基本身没盖好，装修再漂亮也是空中楼阁。"

---

## 三、SFT vs DPO 的本质区别（灵魂拷问题）

> **一句话**："SFT 教'能说话'，DPO 教'说好话'。"
>
> **故事化表达**：SFT 像师父在旁边手把手教菜谱——客人说'要辣的'你就给麻婆豆腐。徒弟学会了**格式**。但他有时会往菜里加'砒霜'（有毒），或虽然熟了但又咸又难吃。DPO 就像你端出 A、B 两盘菜说'选 A 别选 B'——只教偏好，不教答案。
>
> **数据差异**：SFT 是 (Instruction, Output) 一对一；DPO 是 (Prompt, Chosen, Rejected) 三元组，**必须同时有正例和负例**，模型通过对比才能学出"价值观"。

---

## 四、SFT 数据怎么准备？（高频踩坑题）

**70% 取决于数据，30% 取决于算法。** 讲 3 个核心点：

1. **格式选型**：Alpaca（单轮一问一答）vs ShareGPT（多轮对话）。**直接 All in ShareGPT**——单轮也能包装成只有一轮的 conversations，脚本只写一套。

2. **清洗三板斧**：
   - **去重**：精确去重 + MinHash+LSH 模糊去重（"今天天气不错" vs "今天天气真不错" 算相似）；
   - **去"AI 味"**：GPT-4 生成的数据里一堆 "As an AI language model..."，**整条删掉**，否则模型会变成道歉复读机；
   - **关键词过滤**：暴力色情广告。

3. **黄金配比 7-2-1**：70-80% 垂直领域 + 10-20% 通用（防灾难性遗忘）+ 5-10% 自我认知（"我是XX助手"，防止它说"我是 ChatGPT"）。

> **加分话术**："**LIMA 论文证明 1000 条精修数据吊打 5 万条粗制数据**。我宁可花一周精标 500 条，不愿爬 5 万条脏数据。"

## 五、为什么用 LoRA / QLoRA？（技术深度题）

**一句话**："全量微调要更新所有参数，**显存是模型大小的 4 倍以上**，7B 模型单卡根本跑不动；还会'灾难性遗忘'，把旧知识冲掉。"

**LoRA 哲学**：冻结原模型，**只外挂两个小矩阵 A 和 B** 学习新知识。**形象比喻**——模型是一本百科全书，全量微调是把字擦掉重写（费劲、容易写坏）；LoRA 是在书页边角**贴便利贴**（轻松、随时可撕）。数学上 ΔW = A × B，A 和 B 参数量只有原模型的 0.1%-1%。

**QLoRA = 4-bit 量化基座 + LoRA 适配器**。7B 模型原本 14G 显存，4-bit 量化后只占 3.5G，**单卡训练标准答案**。

---

## 六、核心参数口诀（务必背下来）

- **Rank**：简单任务 8，复杂任务 64，**16 是性价比之选**；
- **Alpha = 2 × Rank**（无脑套公式）；
- **Learning Rate**：`2e-4`（LoRA 比全量微调高一个数量级，因为可训参数少）；
- **Target Modules**：**全部线性层**（q/k/v/o + gate/up/down），比只挂 q/v 效果提升明显，显存增加不多；
- **NEFTune**：加 `neftune_noise_alpha=5` 给 Embedding 加噪，**亲测显著提升指令遵循**；
- **Loss 曲线判断**：Loss 不是越低越好。**Loss=1.3 反而比 1.8 差**——它在背训练集。正确判断 = "Loss 下降 + 验证集不反弹 + 真实样本稳定"。

---

## 七、DPO 数据三大剧毒（避雷题）

DPO 对数据极其敏感，**踩一个模型就废**：

1. **长度偏见**：如果 Chosen 总比 Rejected 长，模型会变成话痨——**刻意保留一些"短而精"的 Chosen**；
2. **用 DPO 修事实错误** = 大错特错。**DPO 修的是风格和安全，不是知识**。知识要在 Pre-train 或 SFT 阶段注入；
3. **相似度过高**：Chosen 和 Rejected 只差一两个字 → Loss 接近 0 → **梯度消失学不到东西**，必须保证明显的逻辑/语气差异。

**数据来源**：人工标注（OpenAI/Anthropic 做法，最贵）；**LLM-as-a-Judge**（用 GPT-4 当裁判，**工业界主流**）。

---

## 八、ORPO 为什么值得了解？（加分题）

> "传统流程是 Base → SFT → DPO 两阶段训练，**训两遍、加载 Reference Model、显存翻倍**。ORPO 把 SFT Loss 和 OR Loss 写进同一个公式，**一次训练搞定，且不需要 Reference Model**——7B 模型 DPO 要 30G 显存，ORPO 只要 20G。**单卡玩 DPO 的人，ORPO 是首选。**"

---

## 九、面试官追问时的救命模板

| 追问 | 一句话回应 |
|---|---|
| SFT 数据多少条够 | 1000-10000 条精标，**LIMA 论文已证明质量 > 数量** |
| 怎么评估微调效果 | LLM-as-a-Judge 主观打分 + 业务指标 A/B 测试双轨 |
| 训练 Loss 不降 | 90% 是数据问题——格式错乱、字段没对齐、混进脏数据 |
| 怎么防止灾难性遗忘 | 加 10-20% 通用数据（Alpaca-GPT4）+ 自我认知数据 |
| QLoRA 精度损失大吗 | NF4 量化，**微调任务几乎可忽略**，推理可反量化回 FP16 |

---

**最后一句心法**：讲微调方案，**不要只说"我用过 SFT/DPO"，要说"我为什么选它、放弃了什么、踩过什么坑"**。**对比表 + 故事化比喻 + 量化数据 = 高级感**。
